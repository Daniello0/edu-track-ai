# UI/UX Design Specification: EduTrack AI

> **Источник истины по коду:** [.cursor/rules/project-conventions.mdc](../.cursor/rules/project-conventions.mdc)  
> **Продукт и архитектура:** [main-design.md](./main-design.md)  
> **Схемы компонентов и состояния:** [schemas-design.md](./schemas-design.md) §6–7

## 1. Визуальная концепция

**Стиль:** Soft Minimalism с элементами Glassmorphism.  
**Атмосфера:** «Спокойное обучение» — мягкие пастельные тона, скруглённые углы, лёгкое размытие фона (`backdrop-filter`).  
**Принцип интерфейса:** минимум когнитивной нагрузки; крупные элементы управления, щедрые отступы, фокус на тексте.

**Радиусы скругления:**
- Карточки и инпуты: `16px`
- Модальные окна и крупные панели: `24px`

## 3. Структура страниц

### 3.1. Header (общий компонент)

- **Слева:** логотип (иконка книги + «EduTrack AI»).
- **Справа:**
  - Переключатель темы (`Sun` / `Moon` из Lucide React).
  - Гость: кнопка «Войти» (outline) → модальное окно с `AuthForm` (email/пароль + «Войти через Google»).
  - Авторизованный пользователь: иконка `User` с выпадающим меню (Профиль, Выйти). «Выйти» вызывает `POST /api/auth/logout` и очищает токены.

### 3.2. Главная страница (`main-page`)

- **Фон:** одно размытое органическое пятно пастельного цвета с медленной CSS-анимацией смещения.
- **Центральный блок:**
  - **Поле ввода:** широкий input, мягкая тень, placeholder: «Вставьте ссылку на YouTube видео…».
  - **Блок настроек** (Segmented Control + селекты):
    - Формат: «Рассказ» / «Саммари».
    - Объём саммари (виден только при «Саммари»): «Краткий» / «Средний» / «Полный».
    - Язык: выпадающий список — RU, EN, Оригинал.
    - Тестирование: toggle Вкл/Выкл; при включении — счётчик вопросов (1–10) и вариантов ответа (3–5).
  - **Кнопка:** «Читать» (primary), при наведении — `transform: scale(1.02)`.

### 3.3. Режим чтения (`reader`)

- **Фон:** однотонный кремовый `#FFFDF5` (light) / `#22232A` (dark).
- **Контент:** текст по центру, `max-width: 800px`, шрифт `Lora`.
- **Шапка:** заголовок видео, бейдж категории, дата обработки.
- **Нижняя панель** (sticky):
  - **Авторизованный пользователь** (материал уже в библиотеке): «Перейти к тестам» (primary, если тест включён) или «В библиотеке» (badge), «Удалить» (secondary).
  - **Гость:** «Сохранить в библиотеку» (primary) → модальное окно авторизации; «Удалить» (secondary) — очищает только локальную сессию.

### 3.4. Экран тестирования (`quiz`)

- **Гость:** интерактивное прохождение без серверной проверки; `POST /api/library/:id/quiz/attempts` недоступен до авторизации и `claim-pending`. Верифицированный score и разбор ошибок с правильными ответами — только после сохранения материала в библиотеку (ответ `POST /api/library/:id/quiz/attempts`). До входа клиент **не получает** `correctAnswerIndex` из API.
- **Авторизованный пользователь:** после завершения теста результат отправляется на сервер через `POST /api/library/:id/quiz/attempts`.
- **Layout:** один вопрос на экран, навигация «Назад» / «Далее».
- **Прогресс:** индикатор «Вопрос X из Y».
- **Взаимодействие:** при выборе варианта — мягкая подсветка `rgba(178, 226, 212, 0.4)`.
- **Результат:** процент успеха, список ошибок с правильными ответами (данные из ответа `POST /api/library/:id/quiz/attempts`).
- **Финал (авторизованный):** «Вернуться к материалу» (primary), «В профиль» (secondary).
- **Финал (гость):** «Сохранить в библиотеку» (primary) → auth modal; «Удалить и выйти» (secondary outline). Экран с правильными ответами недоступен — верифицированная оценка появится после claim и прохождения на сервере.

### 3.5. Профиль (`profile`)

- **Дашборд:** две плитки статистики — «Пройдено тестов», «Любимая категория».
- **Список материалов:** карточки с превью YouTube, заголовком, бейджем категории, датой `lastViewedAt` и бейджем статуса.
- **Действие на карточке:** кнопка «Продолжить» → экран чтения.

## 4. Специфические механики

### Сохранение для гостя

1. Гость обрабатывает видео — бэкенд сохраняет полный результат в pending store; клиент получает `pendingId` и публичные данные (quiz без `correctAnswerIndex`). UI хранит их в Zustand и `sessionStorage` (текущая вкладка).
2. Гость нажимает «Сохранить в библиотеку» → модальное окно: «Чтобы не потерять конспект, создайте аккаунт».
3. Авторизация через Firebase (Google или email/пароль) → `POST /api/auth/session`.
4. Автоматический `POST /api/library/claim-pending` с `{ "pendingId": "..." }` из `sessionStorage`.
5. `sessionStorage` очищается; pending store на сервере удаляется; пользователь перенаправляется в reader с `isPersisted: true`.

> **Важно:** до авторизации материал **не записывается в БД**. Закрытие вкладки без входа — потеря данных.

### Состояния загрузки

| Шаг | Отображение |
| :--- | :--- |
| `transcribing` | Пульсирующая иконка «AI» + текст «Получаем транскрипт…» |
| `ai_processing` | Тот же индикатор + текст «Обрабатываем материал…» |
| `completed` | Плавный fade-in экрана чтения |
| `error` | Текст ошибки под формой на главной странице |

### Переходы между экранами

Все смены экранов — `fade-in` 300ms (`opacity` 0 → 1).

## 5. Адаптивность

- **Breakpoint:** `768px`.
- **Мобильная вёрстка:** настройки располагаются под полем ввода (вертикальный стек).
- **Чтение:** `max-width: 100%`, боковые отступы `16px`.
- **Дашборд:** плитки статистики в одну колонку на мобильных.
